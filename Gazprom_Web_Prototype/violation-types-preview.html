Виды нарушений — визуальные варианты

Демо-страница для выбора UX перед реализацией. Задача: хранить устаревшие виды, создавать новые и задавать соответствие «старый → новый», чтобы отчёты и дашборды автоматически группировали исторические нарушения под актуальными видами.

Сейчас в коде: 17 видов зашиты в ViolationTemplates.VIOLATION_TYPES, поле vid у нарушений в актах и реестре. При смене классификатора старые значения «ломают» фильтры и диаграммы. Нужно: отдельная плитка в настройках + механизм миграции.

Вариант A — Split-модалка «Сопоставить виды»

В настройках новая плитка «Виды нарушений». По кнопке «Сопоставить» открывается модалка: слева архивные виды (с количеством нарушений в базе), справа активные. Выбрал пару → стрелка → сохранить. Внизу превью: как изменится столбец в отчёте.

+ Наглядно, быстро для 1–3 замен. Знакомый паттерн (как выбор в модалках визарда).
− Неудобно при массовой перестройке всего классификатора (17+ видов).

Вариант B — Полноэкранный редактор с вкладками

Отдельный экран (как реестр нарушений): вкладки «Активные», «Архив», «Соответствия». На вкладке соответствий — таблица: каждая строка = старый вид, выпадающий список = новый. Удобно для пакетного редактирования всего классификатора.

+ Всё в одном месте, видна полная картина. Хорошо для администратора.
− Менее наглядна связь «слева-справа», чем split-модалка.

Вариант C — Пошаговый мастер миграции

Запускается при обнаружении несопоставленных архивных видов или по кнопке «Обновить классификатор». Шаги: обзор → сопоставление пар → превью отчётов до/после → подтверждение. Снижает риск ошибки при крупной смене нормативки.

+ Безопасно, с превью влияния на дашборды. Подходит для редких, но важных обновлений.
− Дольше для простой замены одного вида; лишние шаги при мелких правках.

Вариант D — Единый список с раскрытием

Все виды в одном списке со статусом. Архивные подсвечены; при раскрытии — поле «заменён на» и кнопка создания нового. Компактно, хорошо на мобильном. Предупреждение сверху, если есть несопоставленные.

+ Минимум экранов, адаптивно, быстрый доступ к редактированию одного вида.
− Сложнее увидеть все пары сразу; массовые операции менее удобны.

Контактные лица

«Нарушения» = реестр формулировок (как сейчас). «Виды нарушений» = отдельный классификатор + архив + соответствия.

Логика данных (для реализации)

  • violationTypes в каталоге: { id, title, status: active|archived, replacedBy? }
  • typeMappings: { fromId → toId } для явных замен
  • При отображении отчётов: resolveVid(v.vid) → актуальный вид
  • В визарде: только active виды в выпадающем списке
  • Старые акты хранят исходный vid; маппинг применяется на чтение (или одноразовая миграция по кнопке)

Рекомендация

Гибрид B + A: основной экран с вкладками (активные / архив / соответствия) — для повседневного управления; кнопка «Сопоставить» открывает split-модалку для наглядной пары. При первом открытии после импорта бэкапа с устаревшими видами — опционально запускать мастер (C) с превью дашборда. Это покрывает и точечные правки, и массовую миграцию без дублирования логики.
