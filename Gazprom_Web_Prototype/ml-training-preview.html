Обучение ML по фото — 3 варианта организации

Макеты для согласования веб-реализации на основе iOS-модуля ViolationMLTrainingViewController и ViolationImageMLService. Все варианты покрывают один и тот же функционал; отличается только навигация и приоритет экранов.

Что уже есть в iOS (переносим в веб):
  • Загрузка фото из актов (инкрементально, с прогрессом)
  • Ручной тест: фото → топ-4 предсказания с % → подтверждение привязки
  • Массовая загрузка с автопривязкой по прогнозу
  • Карточки обучения (проверка/исправление привязок, свайп)
  • Реестр результатов: поиск, сортировка, детализация по нарушению
  • Статистика: фото, нарушения, точность leave-one-out, дата загрузки

Вариант A — «Как в iOS»: один экран-скролл из Настроек

Прямой перенос ViolationMLTrainingViewController: все действия на одной странице — статистика, кнопки управления, тест с фото, карточки предсказаний. Модальные экраны (результаты, карточки, массовая загрузка) открываются поверх.

Плюсы: минимум новой логики навигации; привычно пользователям iOS; быстрый доступ ко всем действиям.
Минусы: длинный скролл на телефоне; на десктопе много пустого места; сложнее для новичка понять порядок действий.

Вариант B — «Пошаговый мастер»: 3 этапа обучения

Разбивает процесс на понятные шаги: 1) Сбор данных → 2) Проверка карточек → 3) Тест и реестр. Каждый шаг — отдельный экран с кнопками «Далее» / «Назад». Подходит для первого запуска и обучения новых пользователей.

Плюсы: понятный workflow; меньше перегруза; естественный onboarding; хорошо на мобильном.
Минусы: больше кликов для опытных; нужна логика «пропустить шаг»; отличается от iOS.
Шаг 1: Данные → Шаг 2: Карточки → Шаг 3: Тест + реестр

Вариант C — «Панель с вкладками»: дашборд для десктопа

Отдельный раздел ML с четырьмя вкладками: Обзор (метрики и быстрые действия), Датасет (реестр привязок), Лаборатория (тест фото), Карточки (очередь проверки). Оптимален для работы на широком экране.

Плюсы: наглядная аналитика; параллельный доступ к разделам; эффективно на планшете/ПК; масштабируется.
Минусы: на узком телефоне вкладки тесные; больше UI-кода; дальше от iOS-оригинала.
