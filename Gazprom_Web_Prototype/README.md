# программа НЕТ — веб-версия

Браузерный каркас приложения «Газпром — акты проверок» (отдельно от iOS-проекта `Gazprom/`).

## Структура

```
программа НЕТ/
├── index.html      # разметка приложения
├── css/app.css     # стили
├── js/app.js       # навигация, мастер акта, часы
├── assets/         # картинки (пока пусто)
├── start_server.command
└── README.md
```

## Запуск в браузере

**Вариант 1** — двойной клик по `start_server.command`, затем откройте:

**http://localhost:3000/**

**Вариант 2** — в терминале:

```bash
cd "Gazprom_Web_Prototype"
python3 dev-server.py
```

После изменения файлов обновите страницу (Cmd+R).

## Постоянный localhost:3000 (автозапуск на macOS)

Чтобы хост `http://localhost:3000/` всегда работал после входа в систему и автоматически
восстанавливался при падении:

```bash
cd "Gazprom_Web_Prototype"
npm run autostart:install
```

Проверка:

```bash
curl -I http://127.0.0.1:3000/
```

Удаление автозапуска:

```bash
cd "Gazprom_Web_Prototype"
npm run autostart:uninstall
```

## Импорт данных с iPhone

Формат файла — тот же, что создаёт `BackupManager` в iOS (JSON, расширение **`.gazprombackup`**).

### Как получить файл с телефона

1. Откройте iOS-приложение → **Настройки** → **Восстановление по дате** (календарь резервных копий).
2. Выберите дату с точкой на календаре → нужную копию в списке.
3. Нажмите **Поделиться** → AirDrop / «Файлы» / почта / iCloud Drive.
4. На Mac откройте веб-версию → **Настройки** → **Выбрать файл** или перетащите файл на карточку.

### В браузере

- Данные сохраняются в **IndexedDB** (остаются после перезагрузки страницы).
- Обновляются разделы: **Главная**, **История**, **Устранение**, мастер акта (организации, комиссия).
- Галочка «Объединить» — как «Восстановить (объединить)» в iOS; без галочки — полная замена.

### Проверка без iPhone

Скачайте демо: `assets/sample-demo.gazprombackup` (ссылка на странице настроек).

## Мастер «Новый акт»

После импорта бэкапа раздел **Новый акт** работает с реальными данными:

1. **Дата и комиссия** — дата, номер (без дубликатов), члены комиссии из справочника  
2. **Организация** — выбор из справочника  
3. **Объекты** — отметка объектов проверки  
4. **Нарушения** — список, фильтр по объекту, просмотр фото (lightbox)  
5. **Описание** — текст и представитель  
6. **Генерация** — сохранение черновика в браузер (Word — позже)

Черновик сохраняется в IndexedDB при каждом шаге «Далее». Кнопка **+ Новый акт** создаёт пустой черновик с новым номером.

## Экспорт Excel / Word (офлайн)

Библиотеки лежат в `assets/vendor/` и кэшируются Service Worker:

- `xlsx.full.min.js` — отчёты и реестр нарушений
- `pizzip.min.js`, `docxtemplater.js` — генерация Word

Первый запуск возможен без интернета после установки PWA (на production HTTPS).
