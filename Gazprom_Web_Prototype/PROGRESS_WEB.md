## Сборка web-222 — формат нарушений в справке

### Переключатели на шаге «Нарушения» (справка)

На шаге 4 мастера справки добавлена панель «Как нарушения попадут в текст Word-документа»:

| Переключатель | Поле в черновике | Эффект |
|---|---|---|
| **Место нарушения** | `violationFormat.includeMesto` | Префикс из поля `mesto`: `Котельная №2: …` |
| **Пункт правил (в скобках)** | `violationFormat.includeRuleRef` | Суффикс из `urlToPravilo` (или `formulaFromRules`): `… (п. 4.1 СП …)` |

Оба переключателя **включены по умолчанию**. Состояние сохраняется в черновике (`draft.violationFormat`), autosave.

**Пример итогового текста:**  
`Котельная №2: Не проведён инструктаж (п. 4.1 СП 12-135-2003)`

### Генерация Word

В `buildSpravkaTemplateData()` маркеры `#ddescrVi#` и `#ddescpitVi#` получают собранный текст через `SpravkaUtils.formatViolationText()`. Отдельные поля `TitleViolatation`, `urlDoc`, `formula` по-прежнему заполняются для шаблонов, где они используются раздельно.

### Превью в карточках

Карточки нарушений на шаге справки показывают тот же формат. Если место или пункт уже в заголовке — дублирующие строки (📍 / 📄) скрываются.

### Файлы

- `js/spravka-utils.js` — `defaultViolationFormat`, `normalizeViolationFormat`, `formatViolationText`, `getViolationRuleRef`
- `js/spravka-wizard.js` — панель переключателей, привязка событий
- `js/wizard-violations-ui.js` — `renderStepExtra`, `formatViolationTitle`, `getViolationFormat`
- `js/doc-generator.js` — применение формата при сборке данных шаблона

## Сборка web-221 — шаблоны справки и UI нарушений

### Шаблон Word для справки (Настройки → Шаблоны Word)

- Раздел **«Справка по ПБ»** рядом с шаблоном акта
- Шаблон по умолчанию: `assets/defaults/Шаблон_справки_ПБ.docx`
- Загрузка своего `.docx`, список сохранённых шаблонов, скачивание пустого шаблона с маркерами
- Справочник маркеров справки (`SPRAVKA_MARKER_GUIDE` в `doc-generator.js`)
- Плитка настроек: статус `акт` / `справка` / `акт+справка`
- Бэкап/импорт: `spravkaTemplate`, `spravkaTemplateName`, `savedSpravkaTemplates`

### Общий UI шага «Нарушения»

Вынесен модуль `js/wizard-violations-ui.js` — единый интерфейс для **акта** и **справки**:

- Карточки нарушений с номером, местом, видом, пунктом правил
- Поиск по нарушениям
- Миниатюры фото, drag-and-drop сортировка
- Секция «Все фото»
- Лайтбокс фото (`js/photo-lightbox.js`)

Подключено: шаг 3 акта (`wizard.js`), шаг 4 справки (`spravka-wizard.js`).

### Субподрядчики и работники в справке

- **Субподрядчики** — выбор из справочника организаций (чипы), строка `subcontractorsList` для Word
- **Работники** — таблица с числовыми полями; организация только из каталога (`orgId` + `orgName`)
- Общий `renderOrganizationChipPicker` в `spravka-utils.js` / `spravka-wizard.js`

## Мастер справки — структура (7 шагов)

| Шаг | Содержание |
|-----|------------|
| 0 | Дата справки |
| 1 | Объекты проверки |
| 2 | Детали объектов (код, линия ГП и т.д.) |
| 3 | Субподрядчики |
| 4 | Нарушения (+ переключатели формата для Word) |
| 5 | Работники (организации + числа ПБ/работников) |
| 6 | Замечания, заключение, генерация Word |
